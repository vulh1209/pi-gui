Design principles to implement

These are the FE-level design moves that improve the implementation without changing the product direction.

implementation-ready guidance

1. Reduce panel chrome height

Collapse the current browser heading and toolbar into a tighter two-row system. Keep the site title + session state visible, but stop spending vertical space on a giant “Browser companion” banner.

2. Use softer browser surface colors

The browser pane can stay light, but it should shift from pure white to a softer neutral paper tone. That lowers the harsh contrast with the dark thread shell.

3. Make thread feedback narrative, not mechanical

Replace generic low-level rows with clearer activity copy: “Opened Google in browser companion”, “Ready for search”, “Typed query”, “Submitted search”.

4. Treat browser as a session, not a widget

Show a tiny session-state chip: pinned / live / same workspace session. This reinforces that the browser is persistent and trusted.

5. Keep browser actions visually grouped

For browser sequences, group related steps into a compact activity cluster or a browser task card, so multiple actions feel like one coherent move.

6. Pull the browser title closer to real page context

Lead with page title + site domain. “Browser companion” should become a small eyebrow, not the main heading.
